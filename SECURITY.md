# 安全策略

## 支持范围

当前仓库处于快速迭代阶段，默认仅维护 `main` 分支的最新版本。

| 版本范围 | 是否支持 |
| --- | --- |
| `main` 分支最新提交 | 支持 |
| 历史快照或旧提交 | 不保证 |

## 报告安全问题

如果你发现了安全相关问题，请不要直接在公开 Issue 中披露完整利用细节。

建议按下面顺序处理：

1. 如果仓库已经启用 GitHub 的私密安全报告功能，请优先使用该渠道。
2. 如果暂未启用，请先通过维护者主页联系仓库维护者：
   - [https://github.com/yinlin712](https://github.com/yinlin712)
3. 在公开渠道中只保留不含敏感细节的摘要信息，等待维护者进一步沟通。

## 希望包含的信息

- 问题影响范围
- 复现步骤
- 可能造成的风险
- 受影响文件、接口或模块
- 如果有缓解建议，也欢迎一并提供

## 响应目标

- 维护者会尽量在 7 个自然日内给出首次响应
- 如果问题确认有效，会协调修复并在合适时机公开说明
- 修复发布前，请尽量避免公开完整利用细节

# 项目分层说明

这个项目刻意分成两部分，请开发时不要混在一起写。

## 1. VS Code 插件端

目录：`src/`、`media/`、`package.json`

职责：

- 在 VS Code 中注册命令和侧边栏
- 展示聊天界面和虚拟形象区域
- 读取编辑器里的当前文件、选中代码等上下文
- 把上下文发给 Python 后端
- 把后端返回的结果显示给用户

这部分是“壳层”，尽量保持轻量。
除非是 VS Code 插件机制要求，否则不要把核心 Agent 逻辑写进这里。

## 2. Python 后端

目录：`backend/`、`requirements.txt`、`environment.yml`

职责：

- 定义后端 API
- 组织提示词
- 调用 Ollama
- 清洗模型输出
- 后续接入工具能力、LoRA 模型切换、实验逻辑

这部分才是项目真正的“智能体核心”。

## 3. 推荐开发原则

- 与 VS Code 生命周期强相关的代码写在插件端
- 与模型、任务编排、推理服务相关的代码写在 Python 后端
- 如果一个功能未来可能被 Web、桌面端或其他 IDE 复用，就优先写在 Python 后端
- 插件端只负责“收集上下文 + 调后端 + 展示结果”

## 4. 给初学者的理解方式

可以把这个项目想成：

- `src/` 是前台接待
- `backend/` 是真正干活的老师

前台接待负责听用户说什么、把信息整理好、把结果展示出来；
真正分析问题、调用模型、生成回答的，是后端老师。
